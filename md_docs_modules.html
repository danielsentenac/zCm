API Modules

Use this page as the single entry point for API modules:

  • High-Level API: Process-oriented helpers that expose config-driven runtime behavior.
  • Low-Level API: Core messaging, registry, socket, and context primitives.

Public structures and where they are implemented

Low-level:

  • zcm_msg_t (opaque), zcm_msg_value_t, zcm_msg_item_type_t, zcm_msg_array_type_t, zcm_msg_status_t, zcm_msg_value_kind_t
    • definitions: include/zcm/zcm_msg.h
    • implementation: src/low-level/zcm_msg.c
  • zcm_node_entry_t, zcm_node_t (opaque), zcm_socket_t (opaque)
    • definitions: include/zcm/zcm_node.h
    • implementation: src/low-level/zcm_node.c

High-level:

  • zcm_proc_data_socket_cfg_t, zcm_proc_type_handler_cfg_t, zcm_proc_runtime_cfg_t, zcm_proc_data_socket_kind_t, zcm_proc_type_arg_kind_t
    • definitions: include/zcm/zcm_proc_runtime.h
    • implementation: src/high-level/zcm_proc_runtime.c
  • zcm_context_t, zcm_broker_t, zcm_proc_t are opaque public handles implemented respectively in:
